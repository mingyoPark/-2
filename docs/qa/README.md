## 🔎 요약

- **테스트 케이스**: 6개 기능 영역, 총 40+ 케이스
- **발견 결함**: 총 9건 (🔴 Critical 3, 🟠 Major 2, 🟡 Minor 4)
  - 대표 결함: SQL Injection(다수), 저장형 XSS, DB 스키마-코드 불일치
- **자동화**: 핵심 사용자 흐름 5종을 Playwright E2E로 자동화 → [`tests/e2e/`](../../tests/e2e)

## 🧭 테스트 접근 방식

1. **명세 기반(Black-box)**: 각 기능의 입력/출력 명세로부터 동등분할·경계값 케이스 도출
2. **탐색적 테스트(Exploratory)**: 비정상 입력·권한 우회·URL 직접 조작으로 결함 헌팅
3. **보안 관점**: 입력값 검증(SQLi/XSS), 인증·인가(세션/CSRF) 점검
4. **회귀 방지**: 핵심 흐름을 자동화하여 변경 시 재검증 비용 최소화
