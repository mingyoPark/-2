# 요구사항 추적 매트릭스 (Requirements Traceability Matrix)

| 요구사항 ID | 요구사항 설명 | 관련 테스트 케이스 | 연결 결함 | 자동화(E2E) |
|-------------|---------------|---------------------|-----------|-------------|
| REQ-01 | 사용자는 회원가입할 수 있다 | TC-AUTH-01,03,05,06 | DEF-006 | auth.spec |
| REQ-02 | 아이디 중복을 사전에 확인할 수 있다 | TC-AUTH-02,03,04 | - | auth.spec |

| REQ-04 | 사용자 입력은 안전하게 처리되어야 한다(보안) | TC-AUTH-12, TC-POST-11,12, TC-LIST-06 | DEF-001,002,004 | board.spec |

## 커버리지 점검

- **요구사항 대비 케이스 매핑률**: 16/16 (100%) — 모든 요구사항에 최소 1개 케이스 연결
- **결함 연결**: 9개 결함 모두 특정 요구사항/케이스에 귀속됨(고아 결함 없음)
- **자동화 커버**: 핵심 요구사항 15/16이 E2E spec과 연결 (REQ-16 공지 열람은 우선순위 낮아 수동)

| 항목 | 사유 | 후속 |
|------|------|------|
| 신고/스크랩 | 화면 미구현(테이블만 존재) | 구현 시 REQ 추가 |
| 소셜 로그인 | UI만 존재 | 구현 시 REQ 추가 |
| 성능/부하 | 범위 외 | 필요 시 별도 계획 |
