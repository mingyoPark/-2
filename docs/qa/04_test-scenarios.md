# 테스트 시나리오 (End-to-End Scenarios)

개별 기능 케이스([02_test-cases.md](02_test-cases.md))와 달리, 실제 사용자 여정을 따라 여러 기능을 연결해 검증하는 **흐름 중심 시나리오**입니다. 자동화([tests/e2e](../../tests/e2e))의 기반이 됩니다.

---

## SCN-01 · 신규 사용자 온보딩 → 첫 글 작성

> 목적: 가입~로그인~글 작성~조회로 이어지는 핵심 획득 경로 검증

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 회원가입 페이지 진입 | 폼 정상 노출 |
| 2 | 아이디 입력 후 중복확인 | "사용 가능한 아이디입니다" |
| 3 | 나머지 정보 입력 후 가입 | 가입 성공 |
| 4 | 가입 계정으로 로그인 | 헤더에 사용자 이름·로그아웃 버튼 |
| 5 | 게시판 → 글쓰기 → 제목/내용/지역/유형 입력 후 등록 | 상세 페이지로 이동, 입력값 표시 |
| 6 | 목록으로 이동 | 방금 작성한 글이 최상단(최신순) |

**연결 케이스**: TC-AUTH-01·03·07, TC-POST-01, TC-LIST-01

## SCN-02 · 커뮤니티 상호작용 (댓글 작성/수정/삭제)

> 목적: 로그인 사용자의 댓글 생명주기 및 권한 검증

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 로그인 후 임의 글 상세 진입 | 댓글 폼 노출 |
| 2 | 댓글 등록 | 목록에 즉시 반영, 작성자=본인 |
| 3 | 본인 댓글 수정 → 저장 | 내용 변경 + "(수정됨)" 표시 |
| 4 | 본인 댓글 삭제(확인) | 댓글 제거 |
| 5 | 로그아웃 후 동일 글 진입 | 댓글 폼 미노출(작성 불가) |

**연결 케이스**: TC-CMT-01·04·05, TC-AUTH-11
**관찰 포인트**: 3단계 "(수정됨)" 표시는 DEF-003(스키마) 수정 후에만 정상.

## SCN-03 · 권한 경계 검증 (타인 글 조작 차단)

> 목적: 인가(Authorization) 우회 방지 — 보안 관점 핵심 시나리오

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 사용자 A 로그인 후 글 작성(pid 확보) | 등록 성공 |
| 2 | 로그아웃 → 사용자 B 로그인 | - |
| 3 | B가 `write.php?pid=<A글>` 접근/수정 시도 | "본인 글이 아니면 수정할 수 없습니다" |
| 4 | B가 `delete.php?pid=<A글>` 직접 호출 | "본인 글만 삭제할 수 있습니다" |
| 5 | 비로그인 상태로 `write.php` 접근 | "회원 전용 게시판입니다" |

**연결 케이스**: TC-POST-05·07·03, TC-CMT-06
**관찰 포인트**: 3~4단계에서 pid에 SQLi 페이로드 병행 시도(DEF-001 확인).

## SCN-04 · 전세보증보험 추천 정확성

> 목적: 조건 분기별 추천 보험사·보증료 계산 정확성 검증

| 단계 | 입력 | 기대 추천 | 예상 보증료 |
|------|------|-----------|-------------|
| 1 | 무주택 아니오 | SGI 서울보증 | deposit×0.01 (기본) |
| 2 | 무주택 예 + 보증지원 이력 있음 | HF 한국주택금융공사 | - |
| 3 | 무주택 예 + 보증금 40,001만원 | HF (4억 초과 분기) | deposit×0.009 |
| 4 | 무주택 예 + 청년 + 중위이하 + 10,000만원 | HUG 주택도시보증공사 | 10000×0.007 = 70.0만원 |
| 5 | 무주택 예 + 서울 지역(그 외 조건 미해당) | HF (서울 분기) | - |

**연결 케이스**: TC-INS-01~08
**관찰 포인트**: 분기 우선순위(무주택 → 이력 → 보증금 → 청년/신혼 → 소득 → 지역) 순서 검증. 음수 보증금 입력 시 처리(DEF-009) 병행 확인.

## SCN-05 · 계약 체크리스트 자가진단

> 목적: 단계 탭 전환과 점수 산출 정확성

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 체크리스트 진입 | '계약 전' 탭 기본 활성 |
| 2 | 탭을 '계약 중'/'계약 후'로 전환 | 해당 섹션만 표시, 버튼 강조 이동 |
| 3 | 전 항목 ⭕ 선택 후 제출 | "모든 항목을 점검하셨습니다" |
| 4 | 일부만 ⭕ 선택 후 제출 | 비율에 따른 경고 메시지 |

**연결 케이스**: TC-CHK-01·02·03·05
**관찰 포인트**: 4단계에서 숨김 탭 required로 제출 실패 여부(DEF-007) 확인.

## SCN-06 · 게시판 탐색 (검색·정렬·페이지네이션)

> 목적: 조회성 기능 통합 검증

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 게시판 진입 | 5개씩, 최신순 |
| 2 | "조회순" 클릭 | 조회수 내림차순 |

| 5 | SQLi 페이로드 검색 | 오류·전체노출 없이 안전 |

**연결 케이스**: TC-LIST-01~06
