## 1. 목적

전세사기 예방 커뮤니티의 핵심 기능이 명세대로 동작하는지 검증하고, 배포 전 결함을 조기에 발견·기록하여 서비스 품질과 사용자 신뢰를 확보한다. 특히 **사용자 입력을 다루는 기능(회원/게시판/보험추천)의 정합성과 보안**을 중점 검증한다.

## 2. 테스트 범위

### 2.1 In-Scope (검증 대상)

| 영역 | 세부 기능 |
|------|-----------|
| 회원 | 회원가입, 아이디 중복확인, 로그인, 로그아웃, 세션 유지 |
| 게시판 | 글 작성/조회/수정/삭제, 목록, 검색, 정렬, 페이지네이션, 조회수 |
| 댓글 | 댓글 작성/수정/삭제, 권한 검증 |
| 공지사항 | 공지 목록/상세 |
| 보험 추천 | 입력 폼 검증, 추천 로직(HUG/SGI/HF), 예상 보증료 계산 |
| 체크리스트 | 단계 탭 전환, 항목 선택, 점수 산출 |
| 비기능 | 입력값 보안(SQLi/XSS), 인증·인가, 기본 반응형/브라우저 호환 |

### 2.2 Out-of-Scope (제외)

- 소셜 로그인(구글/네이버/카카오) — UI만 존재, 미구현
- 무료 법률 자문 예약 — 링크만 존재
- 신고/스크랩 기능 — DB 테이블만 존재, 화면 미구현
- 성능/부하 테스트, 접근성(WCAG) 정밀 검증

## 3. 테스트 전략

| 유형 | 방식 | 도구 |
|------|------|------|
| 기능 테스트 | 명세 기반 수동 테스트 (동등분할·경계값 분석) | 브라우저 |
| 탐색적 테스트 | 비정상 입력·권한 우회·URL 조작 | 브라우저, DevTools |
| 보안 테스트 | SQL Injection / XSS / CSRF / 인가 우회 점검 | 수동 페이로드 |
| 자동화(E2E) | 핵심 사용자 흐름 회귀 자동화 | Playwright |

**우선순위 원칙**: 사용자 데이터·인증·금전(보증료) 관련 기능 > 조회성 기능 > UI 표시.

## 6. 결함 관리 기준

### 심각도(Severity)
| 등급 | 정의 |
|------|------|
| 🔴 Critical | 데이터 유출/손상, 인증 우회, 서비스 중단 |
| 🟠 Major | 핵심 기능 오동작, 우회 가능한 권한 문제 |
| 🟡 Minor | 부분 기능 오류, 유효성 누락 |
| ⚪ Trivial | UI/오탈자/사용성 |

### 우선순위(Priority)
`P1(즉시)` > `P2(차기 배포)` > `P3(여유 시)`

## 7. 진입/종료 조건

- **진입 조건**: 앱이 테스트 환경에서 기동되고, 시드 데이터·테스트 계정이 준비됨
- **종료 조건**: 계획된 테스트 케이스 100% 수행, Critical/Major 결함 0건 또는 처리계획 합의

## 8. 리스크

| 리스크 | 영향 | 대응 |
|--------|------|------|
| 다수의 SQL Injection 취약점 | 데이터 유출/변조 | 결함 등록(P1), prepared statement 전면 적용 권고 |
| 스키마-코드 불일치로 미기동 | 테스트 불가 | 스키마 정정본(`db/01_schema.sql`) 제공 |
| 미구현 기능 다수 | 테스트 범위 혼선 | Out-of-Scope 명시 |
